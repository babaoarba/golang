Functions

Functions

A function is an independent section of code that maps one or more input parameters to one or more output parameters. A functions allow to execute a section of code one or more times. Also, a function allows to break a large job into smaller pieces on which programmers can work independently.

func average(xs []float64) float64 {
    total := 0.0
    for _, v := range xs {
        total += v
    }
    return total / float64(len(xs))
}

func main() {
    vals := []float64{98, 93, 77, 82, 83}
    fmt.Println(average(vals)) // 86.6
}

range over a slice returns two parameters: index and value. The _ tells the compiler to ignore the index.

Variadic functions

func add(args ...int) int {
    total := 0
    for _, v := range args {
        total += v
    }
    return total
}

func main() {
    vals := []int{1, 2, 3}
    fmt.Println(add(vals...)) // 6
}

Clojures

Clojure is a term that indicates a function that access a local or non-local variable .

clojures with local variables

x := 0
increment := func() int {
    x++
    return x
}
fmt.Println(increment())
fmt.Println(increment())

In the case above the increment functin and x variable form the clojure.

func makeEvenGenerator() func() uint {
    i := uint(0)
    return func() (ret uint) {
        ret = i
        i += 2
        return
    }
}

func main() {
    nextEven := makeEvenGenerator()
    fmt.Println(nextEven()) // 0
    fmt.Println(nextEven()) // 2 
    fmt.Println(nextEven()) // 4
}

clojures with non-local variables

func main() {
    for i := 0.0; i < 360.0; i += 45.0 {

        rad := func() float64 {
            return i * math.Pi / 180
        }()
        fmt.Printf("%.2f Deg = %.2f Rad\n", i, rad)
    }
}

In the last example the i var and func has both the scope of makeEvenGenerator and func form a clojure arround i.

Recursion

A function is able to call itself.

func factorial(x uint) uint {
    if x == 0 {
        return 1
    }
    return x * factorial(x-1)
}

func main() {
    fmt.Printf("%d\n", factorial(6)) // 720
}

defer, panic and recover

func main() {
    defer func() {
        str := recover()
        fmt.Println(str) // PANIC
    }()
    panic("PANIC")
}

The defer keyword postpone the execution of func() after the main function finishes. This behaviour is useful to avoid crashing a program when it triggers an error.

panic will terminate the program but recover that is run after panic catch the error and stops the program prom terminating.

Pointers

For an overview on pointers check variables section.

When we pass an argument to the function the argument will be copied.


func zero(x int) {
    x = 0
}
func main() {
    x := 5
    zero(x)
    fmt.Println(x) // x is still 5 
}

To modify the variable we pass his memory address as argument:

func zero(xPtr *int) {
    *xPtr = 0
}
func main() {
    x := 5
    zero(&x)
    fmt.Println(x) // x is 0
}

new

func one(xPtr *int) {
    *xPtr = 1
}
func main() {
    xPtr := new(int)
    one(xPtr)
    fmt.Println(*xPtr) // x is 1
}

Another way to get a pointer is to use the built in function new.
