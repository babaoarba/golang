Error Handling

Error handling

In Go, the traditional way of signaling errors is to return a value of type error when something goes wrong during the execution of the function.

package main

import (
    "errors"
    "math"
    "log"
    "fmt"
)

func div(a, b float64) (res float64, err error) {
    if b == 0 {
        return math.NaN(), errors.New("not allowed to divide by 0")
    }

    return a + b, nil
}

func main() {
    res, err := div(1, 0)
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println(res)
}

The error type is a built-in interface and, therefore must be implemented before it can be used. This is achieved using errors.New()

It is also idiomatic to assign error values to high-level variables so they can be reused throughout a program as needed. 

 var (
      ErrInvalid    = errors.New("invalid argument")
      ErrPermission = errors.New("permission denied")
      ErrExist      = errors.New("file already exists")
      ErrNotExist   = errors.New("file does not exist")
)
