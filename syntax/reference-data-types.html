Reference data types

Reference Data Types

Slices

A slice is a segment of an array. Like arrays slices are indexable and have a length that is allowed to change.

var x []float64             // x has length zero and we can not insert value into x
x := make([]float64, 5)     // x is associated with an underlying array of len 5
x := make([]float64, 5, 10) // x has a length of 5 but hte underlying array has capacity 10

append

slice1 := []int{1,2,3}
slice2 := append(slice1, 4, 5)

The result of slice2 is [1 2 3 4 5]

copy

slice1 := []int{1,2,3} 
slice2 := make([]int, 2)
copy(slice2, slice1)

The copy function will copy slice1 into slice2. Because slice2 has a length of 2, only 1 and 2 will be copy.

Maps

Maps is an unordered collection of key value-pairs.

var x map[string]int

x is a map of string keys and integers values. but if we try to initialize the map with values we get an error:

x["key"] = 10
fmt.Println(x) // panic: runtime error: assignment to entry in nil map

The error is because the map points to a nil value. The map has no underlying structure. To fix the error we use the make function.

x := make(map[string]int) 
x["key"] = 10
fmt.Println(x["key"]) // 10

delete

delete(x, "key") // this will delete the key field from map

Examples

elements := make(map[string]string)
    elements["H"] = "Hydrogen"
    elements["He"] = "Helium"
    elements["Li"] = "Lithium"
    elements["Be"] = "Beryllium"
    elements["B"] = "Boron"
    elements["C"] = "Carbon"
    elements["N"] = "Nitrogen"
    elements["O"] = "Oxygen"
    elements["F"] = "Fluorine"
    elements["Ne"] = "Neon"
    fmt.Println(elements["Li"])
}

// shorter syntax
elements := map[string]string{"H": "Hydrogen",
    "He": "Helium",
    "Li": "Lithium",
    "Be": "Beryllium",
    "B": "Boron",
    "C": "Carbon",
    "N": "Nitrogen",
    "O": "Oxygen",
    "F": "Fluorine",
    "Ne": "Neon",
}

elements := map[string]map[string]string{
        "H": map[string]string{
            "name":  "Hydrogen",
            "state": "gas",
        },
        "He": map[string]string{
            "name":  "Helium",
            "state": "gas",
        },
        "Li": map[string]string{
            "name": "Lithium",
            "state": "solid",
        },
        "Be": map[string]string{
            "name":  "Beryllium",
            "state": "solid",
        },
        "B": map[string]string{
            "name": "Boron",
            "state": "solid",
        },
        "C": map[string]string{
            "name": "Carbon",
            "state": "solid",
        },
        "N": map[string]string{
            "name":  "Nitrogen",
            "state": "gas",
        },
        "O": map[string]string{
            "name": "Oxygen",
            "state": "gas",
        },
        "F": map[string]string{
            "name": "Fluorine",
            "state": "gas",
        },
        "Ne": map[string]string{
            "name":  "Neon",
            "state": "gas",
        },
    }
    if el, ok := elements["Li"]; ok {
        fmt.Println(el["name"], el["state"])
    }

In the last example the map stores as value another map. The if statment is used to check if element Li is part of map.
