Aggregate data types

Aggregate Data Types

Arrays

An array is a numbered sequence of elements of a single type with a fixed length. In Go, they look like this:

var x [5]int

var x [5]int
x[4] = 100     // set the fifth element of an array to 100
fmt.Println(x)

var x [5]float64
x[0] = 98
x[1] = 93
x[2] = 77
x[3] = 82
x[4] = 83

x := [5]float64{ 98, 93, 77, 82, 83 } // short notation
x := [...]float64{ 98, 93, 77, 82, 83 } // the ... creates automatically the index

Structs

A struct is a type that contains named fields.

type Circle struct {
    x float64
    y float64
    r float64
}

var c Circle          // initialize struct with fields set to their default value
c := new(Circle)      // like above but return pointer and allocate memory for all fields
c := Circle{x: 0, y: 0, r: 5} // more readable syntax
c := Circle{0, 0, 5}  // the same as above
c := &Circle{0, 0, 5} // alternative to new keyword

Fields of Struct

Fields can be access with the . operator. A struct’s fields usually represent the has-a relationship (e.g., a Circle has a radius).

fmt.Println(c.x, c.y, c.r) 
c.x=10
c.y=5

Methods

Methods are a special type of function.

func (c *Circle) area() float64 { 
    return math.Pi * c.r*c.r
}

fmt.Println(c.area())

For more informations on methods see OOP section.

Embedded Types

The embedded type represent the is-a relationship. For example, in the code below, the Android is a person. The embedded type is Person and Android can access Person methods.

type Person struct { 
    Name string
}
func (p *Person) Talk() { 
    fmt.Println("Hi, my name is", p.Name)
}

type Android struct { 
    Person 
    Model string 
}

a := new(Android)
a.Person.Talk()
a.Talk() // the same as above
